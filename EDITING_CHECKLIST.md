# EDITING & FACT-CHECK CHECKLIST

## Technical accuracy
- [ ] Every claim cites a primary source; links collected at end.
- [ ] Versions and dates included for standards and products.
- [ ] UI/config names match the product exactly.

## Completeness
- [ ] Procedures include pre-reqs, steps, validation, rollback, owner.
- [ ] Every control has “how to verify” and a success criterion.

## Style
- [ ] Active voice; imperative steps.
- [ ] No filler/marketing clichés.
- [ ] Headings = one idea; lists ≤ 7 bullets.

## Risk/scope clarity
- [ ] Assumptions and limits stated (e.g., tenant size, licenses).
- [ ] Data sensitivity/retention called out if relevant.

## Legal/comms
- [ ] Incident comms avoid blame; include verification steps.
- [ ] Licensing clear for included snippets; standards properly cited.
